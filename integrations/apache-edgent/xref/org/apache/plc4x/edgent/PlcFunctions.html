29  /**
30   * WIP - A plc4x Apache Edgent {@link Supplier} and {@link Consumer} connector factory.
31   * <p>
32   * TODO:
33   * Are there cases where a single logical poll would want to read from 
34   * multiple addrs/data (of different types) from a device and bundle the values
35   * into a single TStream tuple (e.g., a JsonObject)?  How would we support that?
36   * Is there a similar need for writing to multiple addrs/values on a device?
37   * Ah... NOTE: plc4c "batch" requests are coming and will help to address this.
38   * 
39   * <p>
40   * Sample use to read plc device data into an Edgent TStream:
41   * <pre>{@code
42   * PlcConnection plcConnection = new PlcDriverManager().getConnection("s7://192.168.0.1/0/0");
43   * plcConnection.connect();
44   * PlcConnectionAdapter adapter = new PlcConnectionAdapter(plcConnection));
45   * 
46   * DirectProvider dp = new DirectProvider();
47   * Topology top = dp.newTopology();
48   * TStream<Byte> stream = top.poll(PlcFunctions.byteSupplier(adapter, "INPUTS/0", 1, TimeUnit.SECONDS);
49   * stream.print();
50   * dp.submit(top);
51   * }</pre>
52   * 
53   * Sample use to write Edgent TStream data to a plc device:
54   * <pre>{@code
55   * PlcConnection plcConnection = new PlcDriverManager().getConnection("s7://192.168.0.1/0/0");
56   * plcConnection.connect();
57   * PlcConnectionAdapter adapter = new PlcConnectionAdapter(plcConnection);
58   * 
59   * DirectProvider dp = new DirectProvider();
60   * Topology top = dp.newTopology();
61   * 
62   * TStream<Byte> stream = ...
63   * stream.print();
64   * TSink<Byte> sink = stream.sink(PlcFunctions.byteConsumer(adapter, "OUTPUTS/0"));
65   * 
66   * dp.submit(top);
67   * }</pre>
68   * 
69   */
